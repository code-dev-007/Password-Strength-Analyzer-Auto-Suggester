Algorithms & Techniques Used

Shannon Entropy
H = L × log₂(N)

Measures password unpredictability. L = length, N = character pool size. Higher entropy → harder to guess.

Brute-Force Model
T = Nᴸ ÷ 10⁹

Estimates crack time assuming 10⁹ guesses/sec (GPU speed). Uses log₁₀ to handle astronomically large numbers.

Fisher-Yates Shuffle
O(n) unbiased permutation

Guarantees no positional bias in generated passwords. Each permutation has equal probability: 1/n! chance.

SHA-256 Hash (Web Crypto API)
256-bit digest

Cryptographic one-way function. Same input → same output always. Used in secure password storage (with salt + bcrypt in production).

Dictionary Attack Defense
Blacklist ∋ commonPasswords

Checks password against 40+ most commonly breached passwords. Production systems use RockYou.txt (14M+ entries).

Scoring Engine
Score ∈ [0, 100]

Weighted multi-criteria scoring: length, character diversity, entropy, and common-password penalty. Maps to Weak / Fair / Strong / Very Strong.
